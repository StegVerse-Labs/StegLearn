# Screen Environment Model

The StegLearn screen environment is the learner-facing command center for curiosity, creation, explanation, review, receipt generation, and portfolio continuity.

It must not become a passive content feed.

The screen should organize learning while repeatedly returning the learner to the world: building, observing, speaking, drawing, moving, reading, filming, experimenting, caring, and revising.

## Design Goal

The screen environment should help a learner complete the loop:

```text
Wonder
→ Build or Observe
→ Explain
→ Review
→ Receipt
→ Portfolio
```

## Primary Areas

### Wonder Board

A place to capture questions, ideas, observations, and open mysteries.

A wonder may be entered by the learner, parent, or AI scaffold.

### Build or Observe Prompt

A prompt that turns a wonder into action.

The prompt should usually send the learner into a real activity rather than keeping them only on screen.

### Explain Capture

A place for the learner to explain what they noticed, made, thought, changed, or wants to try next.

For younger learners this should support parent transcription, audio, video, drawing, and short quotes.

### Artifact Upload

A place to attach evidence from the learning event.

Artifacts may include photos, drawings, video, audio, screenshots, game maps, code, notes, or parent observations.

### Parent Review

A parent or steward reviews the event, adds context, accepts or rejects the receipt, and may map it to subjects or compliance categories.

### Receipt Timeline

A chronological view of reviewed learning receipts.

This should show learning as continuity rather than isolated assignments.

### Portfolio Map

A growing map of learner questions, artifacts, projects, revisions, interests, and responsibilities.

The portfolio may show patterns but must avoid turning patterns into fixed identity labels.

## Screen Rule

The screen is not the teacher, parent, or destination.

The screen is the organizing surface for a larger learning environment.

## V1 Screens

```text
Learner Home
Wonder Capture
Activity Prompt
Artifact Capture
Explain Capture
Parent Review
Receipt View
Portfolio View
```

## Interaction Principle

The learner should always know what kind of action is being invited:

```text
Ask
Build
Observe
Explain
Review
Continue
Reflect
```

## Non-Capture Boundary

The screen may guide the next step.

It must not define what the learner is, what the learner must become, or which path is the learner's destiny.
