# Learner Home Screen

The learner home screen is the first learner-facing screen in StegLearn.

It should feel like a creative command center, not a school portal.

## Purpose

The home screen helps the learner begin or continue a learning loop.

It should support curiosity, creation, explanation, and continuity.

## Primary Tiles

```text
Ask a Question
Build Something
Tell What Happened
Continue a Project
Look at My Portfolio
Wonder of the Day
```

## Ask a Question

Starts a new wonder.

The learner may type, dictate, draw, or have a parent enter the question.

## Build Something

Starts with action rather than a question.

Useful for learners who think through making first and explaining later.

## Tell What Happened

Lets the learner explain an event that already occurred.

This is important because learning often appears outside planned lessons.

## Continue a Project

Returns to an existing open loop.

Examples:

- unfinished build
- open question
- experiment to repeat
- story in progress
- game map revision
- reading reflection

## Look at My Portfolio

Shows reviewed receipts and artifacts in a learner-friendly way.

The portfolio should celebrate continuity without labeling the learner as a fixed type.

## Wonder of the Day

Offers an optional prompt.

The prompt should be treated as an invitation, not a command.

## Young Learner Support

The home screen should support:

- icons
- audio prompts
- parent assist mode
- short interactions
- low reading load
- touch-first navigation
- large clear actions

## Older Learner Support

The home screen should support:

- project lists
- open questions
- drafts
- artifact uploads
- self-reflection
- subject mapping visibility when appropriate

## Safety Defaults

The learner home screen should not expose public sharing by default.

Sharing, exporting, or posting should require parent review.

## V1 Rule

The home screen should make it possible to start a receipt in less than one minute.
