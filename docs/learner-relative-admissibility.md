# Learner-Relative Admissibility

Admissibility in StegLearn should change as the learner becomes better understood.

A learning event is admissible when the evidence is sufficient for this learner, at this stage, in this mode, with this review authority.

## Core Principle

Admissibility is not one fixed checklist for all learners.

It is a governed relationship between:

```text
learner profile
learning mode
evidence type
review authority
current developmental stage
known communication style
known demonstration style
receipt purpose
```

## Why This Matters

Some learners show understanding through speech.

Some show it through building.

Some show it through drawing, movement, play, game design, imitation, repair, narration, sequencing, music, care tasks, or quiet observation.

If the system only admits written answers, it will miss real learning.

If the system accepts anything as learning, it loses evidentiary meaning.

Learner-relative admissibility preserves both flexibility and review discipline.

## Evidence Modes

Examples of evidence modes:

```text
spoken-explanation
written-explanation
build-artifact
drawing-artifact
video-demonstration
audio-reflection
game-design
physical-performance
care-task
conversation
question-pattern
revision-history
observed-behavior
parent-transcribed-language
```

## Learning Style vs Identity Capture

A learning style is a current evidence pattern.

It is not a fixed identity.

Allowed:

```text
This learner currently demonstrates understanding well through building and spoken explanation.
```

Not allowed:

```text
This learner is only a visual learner.
```

## Admissibility Profile

A learner admissibility profile should record:

```text
preferred evidence modes
accepted evidence modes
emerging evidence modes
modes needing support
review notes
non-capture note
last reviewed date
```

## Dynamic Change

The profile should change through parent review and receipt history.

A mode may move from emerging to accepted after repeated successful receipts.

A mode may need support when evidence is inconsistent or hard to interpret.

## Review Boundary

The parent or steward remains the acceptance authority for profile changes.

AI may suggest profile updates but must not silently change the learner admissibility profile.

## V1 Requirement

V1 should allow the parent to choose an evidence mode during review.

Accepted receipts should update a learner profile summary locally.
