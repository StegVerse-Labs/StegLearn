# Portfolio Model

The StegLearn portfolio is the continuity record formed from accepted learning receipts.

It should show growth without reducing the learner to a fixed profile.

## Purpose

The portfolio preserves evidence over time.

It helps parents, learners, and future tools see patterns across questions, activities, artifacts, explanations, revisions, and responsibilities.

## Portfolio Inputs

The portfolio is built from accepted receipts.

Inputs may include:

- learner questions
- artifacts
- explanations
- parent reviews
- subject mappings
- activity types
- evidence items
- follow-up questions
- project links
- revisions

## Portfolio Views

### Timeline View

Shows receipts in chronological order.

### Project View

Groups receipts around multi-session projects.

### Question View

Shows open, answered, revisited, or transformed questions.

### Artifact View

Shows builds, drawings, videos, game maps, writing, recordings, and other creations.

### Subject View

Shows reviewed subject mappings for homeschool records.

### Growth View

Shows changes in explanation depth, revision behavior, persistence, care, responsibility, and self-direction.

## Pattern Language

The portfolio may describe recurring patterns carefully.

Allowed:

```text
Recent receipts show recurring interest in worldbuilding, game design, and systems questions.
```

Not allowed:

```text
This learner is destined to be a game designer.
```

## Non-Capture Rule

The portfolio records evidence.

It must not become a destiny engine.

Patterns may guide scaffolding, but they must not fix identity.

## Homeschool Role

The portfolio may help organize records for homeschool review.

It should allow exports by date range, subject mapping, project, activity type, or evidence type.

## V1 Portfolio Fields

```text
learner_id
receipt_ids
open_questions
active_projects
accepted_subject_mappings
artifact_index
parent_notes
created_at
updated_at
```

## Privacy Boundary

Portfolio data should be private by default.

Any public sharing or export should require parent review.
