# Admissibility Profile Update Policy

Accepted learning receipts should not automatically force an evidence mode into permanent accepted status.

A parent or steward should decide how the evidence mode updates the learner admissibility profile.

## Purpose

A learner may produce a valid receipt through a mode that is:

```text
already accepted
new and emerging
valid but still needing support
not yet strong enough for profile promotion
```

The receipt can be accepted while the profile update remains cautious.

## Profile Update Options

```text
accepted
emerging
needs-support
no-change
```

## Accepted

Use when the parent decides this evidence mode is reliable for this learner.

The mode is added to accepted and preferred evidence modes.

## Emerging

Use when the parent sees real learning evidence but wants more observations before accepting the mode as established.

The mode is added to emerging evidence modes.

## Needs Support

Use when the event is accepted, but this mode still needs scaffolding, context, translation, or parent interpretation.

The mode is added to modes needing support.

## No Change

Use when the receipt is accepted but should not affect the learner admissibility profile.

## Non-Capture Rule

Profile updates describe current evidence handling.

They must not define what the learner is or limit what the learner may become.

## V1 Behavior

V1 should allow the parent to choose a profile update option during review.

The app should save the choice with the admissibility decision and update the profile accordingly.
