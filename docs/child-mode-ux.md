# Child Mode UX

Child Mode is the learner-facing experience for StegLearn.

It must support young learners, emerging readers, speech-first interaction, touch navigation, visual prompts, and parent-assisted use.

## Purpose

Child Mode helps the learner begin and continue learning loops without requiring adult-style menus, typing, or abstract dashboards.

## Core Design Rules

```text
large actions
few choices per screen
speech-friendly
icon-supported
parent assist available
no public sharing
no hidden monetization
no infinite feed
```

## Primary Actions

```text
Ask
Build
Watch
Tell
Continue
Look Back
```

## Ask

The learner speaks, types, draws, or parent-enters a question.

## Build

The learner starts a making activity.

This may include physical builds, drawings, game design, code, stories, experiments, or outdoor observations.

## Watch

The learner plays a parent-approved or local video.

Playback should lead toward explanation, application, or reflection.

## Tell

The learner records or dictates what happened.

## Continue

The learner resumes an open project, question, or receipt draft.

## Look Back

The learner sees accepted portfolio items in a simple visual timeline.

## Read-Aloud Support

Any text prompt shown to the learner should be readable aloud by the system or parent.

## Friction Rule

The learner should be able to start a new wonder, record a spoken explanation, or attach an artifact with minimal steps.

## Safety Rule

Child Mode should never expose public posting, account settings, legal export, or raw privacy controls.

Those belong in Parent Mode.
