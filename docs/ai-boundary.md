# AI Boundary

AI support in StegLearn is optional and review-bound.

The learning loop must work without AI dependency.

## Purpose

AI may help the parent and learner move from question to activity, explanation, receipt, and follow-up.

AI must not become the hidden authority over the learner.

## Allowed AI Uses

```text
suggest activity paths
ask follow-up questions
draft receipt summaries
summarize parent-reviewed transcripts
suggest subject mappings
organize portfolio patterns
prepare export drafts
```

## Disallowed AI Uses

```text
final learner evaluation
fixed identity labeling
unreviewed public sharing
automated compliance certification
hidden persuasion
ranking the learner against others
forcing a curriculum path
```

## Review Requirement

AI outputs that affect receipts, subject mappings, parent notes, or portfolio summaries require parent review.

## Evidence Boundary

AI summaries are not raw evidence.

The system must preserve the underlying parent note, learner quote, transcript, artifact, or receipt that supports the summary.

## Offline Boundary

The app should remain useful without AI.

AI can improve scaffolding, but it must not be required for basic learning capture.

## V1 Rule

In v1, AI may draft suggestions only.

The parent accepts, edits, rejects, or ignores them.
