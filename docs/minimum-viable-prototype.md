# Minimum Viable Prototype

The StegLearn MVP should prove one complete learner loop using an on-screen environment with speech and video support.

It should not attempt to become a full school, curriculum engine, or social platform in the first version.

## MVP Goal

A learner can ask or speak a wonder, engage with an activity, attach or record evidence, explain what happened, and send the session to a parent for review.

The parent can review, accept, and generate a learning receipt.

## MVP Flow

```text
Learner Home
→ Ask or Speak Wonder
→ Choose Activity Type
→ Watch, Build, Observe, or Create
→ Attach Artifact or Record Explanation
→ Send to Parent Review
→ Parent Accepts Receipt
→ Portfolio Saves Receipt
```

## Required Screens

```text
Child Home
Wonder Capture
Activity Prompt
Video Playback
Speech Record
Artifact Attach
Parent Review
Receipt View
Portfolio Timeline
```

## Required Capabilities

```text
speech recording
speech transcript draft
video playback
media artifact attachment
parent transcript review
receipt generation
local receipt storage
privacy status marking
```

## Not Required in MVP

```text
full curriculum generation
public sharing
multi-family accounts
marketplace content
legal compliance certification
automated grading
social feed
gamified reward loops
```

## Success Criteria

The MVP is successful when it can generate a valid receipt for at least these cases:

- spoken question
- watched video with explanation
- physical build with photo evidence
- game design reflection
- parent-transcribed early language event

## Implementation Preference

Prefer local-first storage and parent-controlled export.

AI scaffolding may be added after the core loop works without AI dependency.

## Non-Capture Boundary

The MVP must not introduce ranking, learner labels, streak pressure, or algorithmic identity profiles.
