# Safety and Privacy

StegLearn may handle child learning records, family context, audio, video, images, artifacts, and parent notes.

Safety and privacy must be built into the system from the start.

## Core Rule

Private by default.

Public only by explicit parent or steward action.

## Child Identity

The system should avoid requiring public names, school names, exact locations, or public identifiers.

Local learner identifiers are preferred.

## Location Safety

The system should avoid exposing:

- home address
- school location
- regular walking routes
- real-time whereabouts
- recognizable private spaces
- metadata that reveals location

## Media Safety

Photos, videos, audio, drawings, screenshots, and game artifacts should remain private unless parent-approved for export or sharing.

## Public Sharing

Public posting should not be a default learner action.

Any public sharing flow should require parent review.

## AI Data Boundary

AI assistance should use the minimum information needed for the task.

AI-generated summaries should distinguish between learner statements, parent notes, artifact descriptions, and inferences.

## Parent Control

Parents or stewards should be able to:

- review receipts
- edit parent notes
- reject receipts
- mark records private
- export records
- delete draft sessions
- control sharing
- inspect AI-generated content

## Homeschool Records

Homeschool records may need to be preserved, exported, or reviewed.

The system should help organize records while avoiding unnecessary exposure of private family data.

## Data Minimization

Do not collect data merely because it may be useful later.

Collect what supports the learning receipt, portfolio continuity, safety, and parent-reviewed records.

## V1 Safety Defaults

```text
private records by default
no public posting from learner screen
parent review before export
local learner IDs
clear separation between learner quote and AI summary
no location publication by default
```

## Non-Capture Safety

Safety systems should protect the learner without turning the learner into a surveillance object.

The goal is stewardship, not constant monitoring.
